Para empezar, vamos a describir los problemas que debemos solucionar:
El primer ejercicio denominado "Secuencia introductoria-2", del cuál aprenderemos a crear ramas en Git.
El segundo ejercicio denominado "Push y Pull-1", del cuál nos intreducirá a la clonación.

Empezaremos explicando el primer ejercicio:

Como podemos ver, hemos creado una rama denominada "bugFix" [con git branch (nombre de la rama)], ya que es la que nos pide el ejercicio
Lo siguiente será utilizar un 'git checkout' + (nombre de la rama) para que nos situemos en la rama que queremos. En este caso al utilizar 'git checkout bugFixed' nos pondremos en la rama de bugFixed y terminaremos el ejercicio.

Con esto realizado hemos aprendido a utilizar git branch (El cuál nos servirá para crear ramas para nuestros trabajos), git checkout (Del cuál hemos aprendido para situarnos en la rama que queremos para avanzar en nuestros proyectos).



Ahora daremos paso a realizar el segundo ejercicio 'Push y Pull-1':
En este caso solo necesitaremos utilizar 1 comando git, pero lo más importante de este ejercicio es saber su funcionalidad.
Por lo que se puede leer en el foro de la página donderealizamos las actividades, sirve para permitir commits de un lado a otro, lo que quiere decir que son copias que pueden estar en x ordenadores y que se utilizarían después para terminar el proyecto que se tenga pensado.

Como podemos ver solo nos ha hecho falta utilizar 'git clone' y se ha duplicado el repositorio que existía anteriormente.
